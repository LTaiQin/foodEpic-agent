# Project Onboarding Command

Please help me understand and continue working on the project at: $ARGUMENTS

## Onboarding Instructions

1. **Read documentation first** - Understand project overview and status
2. **Analyze project structure** - Identify key modules and patterns
3. **Assess current status** - Determine completed and pending work
4. **Start productive work** - Begin with quick wins and build momentum

## Required Reading Order

### Layer 1: Project Status
1. Project overview/README
2. Handoff documents (if available)
3. Implementation plans
4. Status reports

### Layer 2: Code Structure
1. Directory layout
2. Main entry points
3. Key modules
4. Configuration files

### Layer 3: Recent Activity
1. Git history
2. Recent commits
3. Active branches
4. Pending changes

## Communication Protocol

- **Language**: Use Chinese for communication (unless specified otherwise)
- **Status reports**: Provide concise updates after each change
- **Format**: "改了什么、怎么验证、下一步做什么"
- **Blocking issues**: Report immediately when stuck

## Work Approach

1. **Start with documentation** - Read before writing
2. **Verify each change** - Test before committing
3. **Update as you go** - Keep documentation current
4. **Communicate progress** - Regular status updates
5. **Ask when blocked** - Don't waste time on unclear requirements

## Output

Provide:
- Project understanding summary
- Current status assessment
- Prioritized task list
- Immediate action plan
- Progress updates

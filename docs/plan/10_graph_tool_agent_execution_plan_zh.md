# 10 图谱工具型 Agent 实施主线

## 10.1 文档目的

这份文档不是补充说明，而是后续主线实现约束。

目标只有一个：

- 把当前系统推进成一个真实的、会主动调用工具的厨房视频 agent。

这里的“真实”指：

- 不是把整理好的内容主动塞给大模型。
- 不是只做一次检索后直接让模型猜答案。
- 不是把 agent 退化成“多模态问答 prompt 包装器”。

---

## 10.2 强约束

### 10.2.1 不主动喂内容

默认情况下，不允许主动输入给大模型的内容：

- 全量帧描述。
- 全量视频摘要。
- 人工整理好的题目证据包。
- 预先选好的关键帧集合。

默认只允许给大模型：

- 问题与选项。
- 工具列表与工具 schema。
- 当前工作记忆。
- 上一步工具返回。

也就是说，大模型面对的是“环境”，不是“现成答案材料”。

### 10.2.2 图谱只是记忆层，不是证据终点

图谱里存的是结构化记忆，不是最终证据。

系统必须始终保留原始数据回查能力：

- 按时间点抽帧。
- 按时间段稀疏抽帧。
- 对 bbox 画框保留上下文。
- 局部放大。
- OCR。
- 候选时间窗口重新观察。

图谱节点必须能反查到：

- 原始视频。
- 对应时间段。
- 对应帧。
- 对应 region。

### 10.2.3 图谱中只能保存评测允许的信息

允许进入图谱：

- 原视频、音频及其派生证据。
- 自动抽帧结果。
- 自动区域图。
- OCR 结果。
- 自动提取的对象、动作、位置、状态、步骤、时间事件。
- agent 自己在执行中写回的观察。

禁止进入图谱：

- benchmark 标准答案。
- 标注侧隐藏真值。
- 任何测试时本不应直接获得的信息。

---

## 10.3 目标系统形态

最终系统流程必须是：

`问题 -> planner -> 工具调用 -> 工作记忆更新 -> 缺证据判断 -> 原始数据回查/图谱再检索 -> 最终回答`

而不是：

`问题 -> prompt -> 模型 -> 直接答案`

系统至少应有四层：

1. Memory Layer：图谱与持久化记忆。
2. Tool Layer：检索、视频、视觉、写回工具。
3. Agent Layer：planner、executor、working memory、verifier。
4. Eval Layer：任务族评测、失败分析、成本统计。

---

## 10.4 大模型在系统中的职责

大模型的职责应被严格限制为：

1. 决定下一步需要什么证据。
2. 选择合适工具。
3. 判断当前证据是否足够。
4. 在证据充足时输出答案。

大模型不应直接承担：

- 离线知识灌输后的直接猜题。
- 不经过工具的长视频理解。
- 用大 prompt 替代检索和证据回查。

---

## 10.5 工作记忆规范

建议统一保持以下最小状态：

```json
{
  "video_id": "...",
  "task_family": "...",
  "question": "...",
  "choices": ["..."],
  "hypotheses": [],
  "evidence": [],
  "visited_times": [],
  "visited_nodes": [],
  "artifacts": [],
  "open_questions": [],
  "confidence": 0.0
}
```

要求：

- 每一步只允许基于当前工作记忆和工具结果继续决策。
- 不允许在 planner 中绕过工具直接做大量题外推断。

---

## 10.6 工具系统约束

### 10.6.1 图谱检索工具

必须长期保留：

- `query_time`
- `query_object`
- `query_event`
- `get_neighbors`

应继续补齐：

- `query_state`
- `query_location`
- `query_ocr`
- `query_region`

### 10.6.2 原始视频访问工具

必须长期保留：

- `extract_frame_at_time`
- `extract_frames_for_range`
- `render_bbox_overlay`
- `extract_region_with_context`

应继续补齐：

- `sample_sparse_frames`
- `render_bbox_sequence`
- `run_ocr_on_region`
- `run_ocr_on_image`
- `extract_audio_clip`
- `detect_audio_peaks`

### 10.6.3 图谱写回工具

这是 agent 和普通问答器拉开差距的关键层。

必须支持把新观察写回外部记忆，例如：

- `write_observation`
- `write_frame_observation`
- `write_region_observation`
- `write_ocr_reading`
- `write_motion_hypothesis`
- `write_timeline_summary`

原则：

- 新观察必须带时间、来源、证据路径、置信度。
- 写回后必须可再次被检索，而不是只存在 prompt 内。

## 10.7 执行循环规范

每个问题都应按下面的标准循环执行。

### Step 1：问题解析

解析出：

- 题型。
- 关键对象。
- 关键时间。
- 是否涉及状态变化。
- 是否涉及位置/方位。
- 是否涉及 OCR/称重。
- 是否需要跨时间比较。

### Step 2：初始规划

planner 输出：

- 当前假设。
- 首个工具调用。
- 调这个工具的原因。

### Step 3：低成本检索优先

优先用图谱检索：

- 时间窗口。
- 对象关键词。
- 事件类型。
- 已有步骤、食材、轨迹或读数节点。

### Step 4：证据不足判断

若当前证据不足，必须指出缺的到底是什么：

- 缺局部时间证据。
- 缺空间定位。
- 缺连续轨迹。
- 缺读数。
- 缺状态变化比较。

### Step 5：原始数据回查

只有在结构化记忆不够时，才进入：

- 稀疏抽帧。
- 画框。
- 局部放大。
- OCR。
- 局部时间段再采样。

### Step 6：工作记忆/图谱写回

把工具新得到的观察写回：

- 工作记忆。
- 图谱节点。
- 时间线摘要。

### Step 7：最终作答

只有当证据足够区分选项时，才允许：

- `finish`
- 输出答案编号。
- 输出证据链。
- 输出工具轨迹。

## 10.8 与研究意义和应用价值的绑定

### 10.8.1 什么任务值得做

只有同时满足下面两点的任务才进入主线：

- 有研究意义：能体现长时序记忆、多模态对齐、工具调用、证据约束或可解释推理。
- 有应用价值：能服务厨房过程记录、营养追踪、步骤检查、证据回放、辅助记忆或教学复盘。

### 10.8.2 优先问题族

优先推进：

- recipe step / multi-step localization
- ingredient add / order / weight
- nutrition change / estimation
- object movement / stationary localization
- fixture / gaze / interaction

这些任务的共同点是：

- 需要证据，不适合纯猜。
- 有明确的厨房应用价值。
- 能明显体现 agent 相比普通大模型的优势。

## 10.9 分阶段实施路线

### Phase G1：记忆层稳固

目标：

- 让单视频图谱真正成为可回查的长期记忆。

任务：

- 完善节点到原始证据的追溯字段。
- 增加 region / OCR / motion / nutrition 相关节点。
- 增加图谱一致性检查脚本。

完成标准：

- 任一关键节点都能回到原始视频或图像证据。
- 图谱构建和写回过程稳定可复现。

### Phase G2：原始数据工具完备

目标：

- 让 agent 真能自己“去看”视频，而不是依赖预先整理好的材料。

任务：

- 完善抽帧、稀疏采样、画框、局部放大、OCR、音频峰值工具。
- 规范工具返回的结构化格式。

完成标准：

- 给定视频与时间窗口，agent 可以自己生成证据图并继续检索。

### Phase G3：任务专用工具链

目标：

- 对真正有研究价值的任务族建立专用推理链，而不是全靠统一 prompt。

任务：

- object motion：显式比较时间点、统计移动次数、判断静止位置。
- nutrition：根据 ingredient event 计算营养增量。
- weight：结合称量事件与视觉/OCR 校验。
- fixture/gaze：结合局部视觉和空间先验定位。

完成标准：

- 每类任务都有明确的工具链与失败类型，而不是模糊的一次性问答。

### Phase G4：从“会调用工具”升级到“会写回知识”

目标：

- 让 agent 具备外部化思考与增量记忆能力。

任务：

- 观察写回图谱。
- 局部时间线写回。
- 对象运动假设写回。
- OCR/重量读数写回。

完成标准：

- 后续问题可复用前面生成的观察，而不是每题重来。

### Phase G5：正式评测与论文结果

目标：

- 形成可用于论文汇报的真实 agent 实验结果。

任务：

- 按任务族做 baseline 对比。
- 统计 accuracy、evidence correctness、tool-use quality、token cost、latency。
- 做 failure analysis 和 ablation。

完成标准：

- 能清楚回答“为什么 agent 比普通 LLM/普通 baseline 更好”。

## 10.10 与普通大模型问答的本质区别

最终系统必须能体现以下差异：

- 普通模型：看到输入后直接回答。
- 本系统：先检索、再观察、再写回、再回答。

真正的优势不应只是：

- prompt 更长。
- 给更多图片。
- 多试几次。

真正的优势应是：

- 能在长视频里缩小搜索空间。
- 能保留长期记忆。
- 能回查原始证据。
- 能把新观察沉淀成可复用知识。
- 能输出可核查的证据链。
